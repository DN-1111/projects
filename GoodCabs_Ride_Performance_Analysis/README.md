-> From Rides to Insights: Goodcabs Performance Analysis

->  Overview
This project analyzes business performance for GoodCabs, a fictional ride-hailing startup, using SQL and Tableau. The goal was to uncover key business drivers such as repeat passenger rates, trip volume, customer satisfaction, and regional performance.

->  Tools Used
- SQL (PostgreSQL/MySQL)
- Tableau Public
- PowerPoint

->  Objectives
- Understand trip patterns and passenger behavior
- Measure repeat passenger rate (RPR) across cities
- Identify underperforming zones and potential improvement areas
- Deliver clear and actionable insights through dashboards and presentations
  

->  Project Structure

| Folder         | Description                                 |
|----------------|---------------------------------------------|
| `SQL/`         | SQL scripts for data analysis and KPIs      |
| `Presentation/`| PPT slides with summary and recommendations |
| `Visualizations/` | Tableau dashboard screenshot             |
| `Data/`        | Primary ,secondary and ad-hoc questions            |

->  Key Business Insights
- Identified underperforming cities for targeted operational improvement
- Revealed that weekend demand was significantly higher in urban hubs
- Flagged a dip in RPR in specific cities, prompting customer retention recommendations
- Found rating gaps across city types, suggesting potential service quality issues
- Provided clear, visual recommendations to boost both revenue and repeat usage

->  What I Learned
- Creating KPIs using CTEs, window functions, and joins in SQL
- Designing effective dashboards to tell data stories
- Translating complex metrics into executive-level insights
